1. What is the difference between invoking and creating a function?

    2. Creating a function means defining it in the code. By adding the word "function" to the beginning of the function, you create the function. Invoking a function means running it. This you can accomplish, without calling a function a function.

  2. Explain the syntax regarding semicolons and code blocks.

    3. Semicolons are used to define standalone statements, while code blocks are used to group together large statements.

  3. What happens when we try to store values from a function when it doesn't return any?

    4. The function will just return "undefined", as the function will not return anything.

  4. How do we separate parameters from each other when creating a function?

    5. When adding parameters to a function, you write them in parentheses after the function title, and separate them by using commas.

  5. Explain how parameter names are related during the invocation and creation of a function.

    6. When creating a function, parameter names are placeholders for values to be later defined. In invoking functions, the arguments are actual values, that are matched to the parameters by position not by name.

  6. Explain why a parameter is undefined when a function call is missing a parameter.

    7. Any parameter that is undefined through the function will automatically become undefined.

  7. Explain what happens when we use recursive functions.

    8. A recursive function is a function that calls itself. When we use recursive functions, the function will keep calling itself repeatedly, losing value along the way, and once it reaches the base case it will cease the recursion.
